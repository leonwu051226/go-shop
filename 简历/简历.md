## 项目经历

### 高并发分布式秒杀系统（个人项目）

**项目时间**：2026.03 - 2026.05  
**项目地址**：github.com/zhangsan/seckill-system  
**技术栈**：Go 1.26、Gin、GORM、MySQL 8.0、Redis 7.0、RabbitMQ 3、Docker Compose

#### 项目简介

设计并实现了一套完整的电商秒杀系统，核心解决高并发场景下的**库存超卖**、**重复下单**、**流量突增**三大难题。系统采用**分层架构**，通过 Redis 原子扣减 + RabbitMQ 异步削峰 + 数据库乐观锁兜底，实现高可用、高并发的秒杀业务流程。

#### 我的职责

- 独立完成系统架构设计、数据库设计、接口开发及全流程联调
- 负责高并发核心链路：库存预热、原子扣减、异步下单、事务落库
- 使用 Docker Compose 编排 MySQL、Redis、RabbitMQ 开发环境

#### 核心设计与技术亮点

**1. 分层架构与工程规范**

- 采用 `cmd / internal / pkg` 标准 Go 项目结构，职责边界清晰
- API -> Service -> Repository -> Model 四层分离，便于单元测试与后续扩展
- 统一响应封装、JWT 认证中间件、配置中心化管理（Viper）

**2. Redis + Lua 原子脚本：防超卖与防重**

- 秒杀活动开始前，通过 `PreheatStock()` 将 MySQL 库存预热到 Redis
- 使用 **Lua 脚本**在 Redis 服务端原子执行：
  - 检查用户是否已购买（`SISMEMBER`）
  - 检查库存是否充足（`GET`）
  - 扣减库存（`DECR`）
  - 记录用户购买标记（`SADD`）
- 整个判断-扣减-记录流程在 Redis 单线程中原子完成，**彻底杜绝超卖和重复购买**

**3. RabbitMQ 异步下单：削峰填谷**

- Lua 脚本返回成功后，立即生成唯一 OrderID，将订单信息序列化为 JSON 投递到 RabbitMQ
- 前端瞬时返回 `"status": "queuing"`，实现**请求快速响应**
- 消费者侧以独立 Goroutine 持续监听队列，将峰值流量平滑到数据库可承受的范围

**4. 事务安全落库 + 幂等性保障**

- 消费者端使用 **GORM Transaction** 包裹两步操作：
  1. 乐观锁扣减数据库库存：`UPDATE seckill_products SET stock = stock - 1 WHERE id = ? AND stock > 0`
  2. 插入订单记录
- 数据库层设置**双重唯一索引**保障幂等性：
  - `order_no` 唯一索引（防止同一条 MQ 消息重复消费）
  - `user_id + seckill_product_id` 联合唯一索引（防止业务层面重复下单）
- 手动 ACK/NACK 机制：成功则确认，重复消息则 Ack 丢弃，系统错误则 Nack 重新入队重试

**5. 基础设施与部署**

- `docker-compose.yml` 一键编排 MySQL 8.0 + Redis 7.0 + RabbitMQ 3（Management）
- 开发环境端口隔离：MySQL 3308、Redis 6380，避免与本地服务冲突

#### 项目成果

- 完整实现用户注册/登录（JWT）、秒杀商品列表、抢购下单、异步落库全链路
- 通过 Redis Lua 脚本 + 乐观锁 + 联合唯一索引，**从缓存层到数据库层形成多级防超卖体系**
